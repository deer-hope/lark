上行协议



消息协议使用文本协议，整个协议由两个层次构成，一个是消息头部，一个是消息的主体（body）。

消息头部是所有协议必须上传的字段，缺少任何一个字段都会产生系统级错误。

消息头部由一下四个字段组成：
String transactionId;
String type;
Integer version;
Long time;

解释：
transactionId：是用于全流程跟踪的关键字段，建议每次客户端主动发送一个消息的时候，都产生一个uuid填入transactionId，
如果有必要最好将其记录到日志中。

type：消息类型，也就是协议类型，用于区分消息的作用，当前服务端已经实现的消息类型有：login，heartbeat，listContact，singleChat

version：协议版本号，注意这个版本是指的这个type协议的版本，并非全局大版本号。也即是说，type不同，version的值就可能不同。
当前所有协议的版本号都是1，随着时间的推移，有些协议的版本号会向上涨。

time：客户端发送消息时的时间戳，是1970年1月1日零点到当前时间的毫秒数，注意是毫秒。

举一个完整消息示例：

{
    "body": {
        "authType": 1,
        "clientIp": "192.168.1.2",
        "clientPort": 6023,
        "clientVersion": "1.0.0",
        "deviceNumber": "123456789",
        "netType": "wifi",
        "password": "huwei",
        "platform": "web",
        "userName": "huwei"
    },
    "time": 1523979857996,
    "transactionId": "473becfc-9b6f-4e58-a59c-22b3db2979da",
    "type": "login",
    "version": 1
}


这个一个完整的login协议，标红的就是协议头部字段，后面将不会再介绍。body里面是具体的业务字段


下行协议

下行协议也是两层模式，一个是头部，一个是主体（body）
下行协议的头部包含如下字段：

String server;
String transactionId;
Stringc;
Integer version;
Long time;
int statusCode;
String statusDescription;

server：服务器的名字，当前取值lark-1.0.0。
transactionId：如果此下行协议对应了一个上行请求，那么这个transactionId和上行的transactionId是相同的值，这个是对服务器的强制约束。
如果没有对应的上行请求，那么这个值有可能是服务端产生的一个随机值
type：和上行协议的解释相同。下行type和上行type一般相同，如果没有对应的上行，那么type将会有单独的解释
version：版本号，一般和上行相同，如果没有对应的上行，那么这个版本号会有单独的解释，依赖于type字段
time：服务端产生的时间戳
statusCode：状态码字段，用于表明当前消息请求的处理结果
statusDescription：状态码解释字段


状态码约定：为通用状态码和业务状态码
通用状态码如下：
public static int success = 1;
public static int parameterError = 2;

public static int systemError = 3;
public static int systemBusy = 4;//系统繁忙，稍后可以重试
public static int frequencyLimitation = 5;//访问频率太快，稍后可以重试
public static int flowLimitation = 6;//流量限制，稍后可以重试

public static int ipBlacklist = 7;//命中了ip黑名单
public static int macBlacklist = 8;//命中了mac黑名单

//10-40段专门划分用于登录相关的错误码，因为登录认证相关的错误码所有的协议都会引用
public static int notLogin = 10;//用户未登录
public static int ticketTimeout = 11;//票据过期
public static int ticketError = 12;//票据非法
