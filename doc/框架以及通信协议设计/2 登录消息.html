1、消息请求示例：

{
    "body": {
        "authType": 1,
        "clientIp": "192.168.1.2",
        "clientPort": 6023,
        "clientVersion": "1.0.0",
        "deviceNumber": "123456789",
        "netType": "wifi",
        "password": "huwei",
        "platform": "web",
        "userName": "huwei"
    },
    "time": 1523979857996,
    "transactionId": "473becfc-9b6f-4e58-a59c-22b3db2979da",
    "type": "login",
    "version": 1
}

authType：认证类型，取值为整数。当前直接填1.含义：表示指明服务端对用户如何进行认证，这个主要是因为有可能很多外部用户接入进来，
那么后台可能需要调用外部接口去认证用户，所以需要此字段。

clientIp：客户端ip，当建立连接后，客户端可以获取到自己用于连接的ip地址
clientPort：客户端端口，当建立连接后，客户端可以获取到自己的端口号，这两个字段主要是可以用于帮助服务端去发现，两个用户是否
来自同一个内网。

clientVersion：客户端版本号，这个是客户端自己定义的版本号，一般是规划客户端的产品经理来定义这个字段的值。
deviceNumber：设备号，用于表示设备的序列号，无论客户端采用什么方式来获取设备号，服务端对其要求是
同一个终端（硬件），设备号相同，不用终端，设备号不用。最好是用户升级客户端，重装客户端后，同一个终端的设备号码
都不变。

netType：当前使用的网络类型，2G，3G，4G，5G，wifi

userName：用户名
password：用户密码
platform：客户端运行平台，ios，android，pc，mweb，pcweb

2、消息响应示例：
{
    "body": {
        "ticket": "DEEI6dBBmfZ2il56jrqTEXfaIqQxjEbT9Y8/pS8Rt9s=",
        "userId": "huwei"
    },
    "server": "lark-1.0.0",
    "statusCode": 1,
    "time": 1528526790252,
    "transactionId": "473becfc-9b6f-4e58-a59c-22b3db2979da",
    "type": "login",
    "version": 1
}

userId：用户的id，注意这个userId是系统内部用于数据流转关联用的id，他和登录是用的userName不一定相同，也有可能相同
ticket：用户的登录凭据，当用户认证通过后，就会得到一个登录凭据，这个凭据将会用于后续所有的协议中
